linux下四种IO模型

2019-09-16

（1）阻塞模型

默认情况下，所有IO在进行“read”,“recv”操作时，都
会是阻塞模式：没有下一步数据的情况下，进程会进入
浅睡眠状态，不能继续运行

（2）非阻塞模型

将“read”,”recv”操作设置为不等待，如果有数据，就接收
如果没有数据，继续往下执行，进程不会进入到睡眠状态

设置文件属性—>在文件操作过程中修改文件属性

#include <unistd.h>

#include <fcntl.h>
int fcntl(int fildes, int cmd, …);
fildes —->被操作的文件描述符
cmd —->操作命令
F_GETFL ：获取文件属性
F_SETFL ：设置文件属性
… —->具体的命令方式，根据cmd的参数而改变

O_NONBLOCK  ：文件IO为非阻塞

彩蛋哦要是能发现就给你一张卡哟 木啊 给最爱的京京呐

获取当前文件属性

long statu = fcntl(fd,F_GETFL);

添加非阻塞属性

statu |= O_NONBLOCK;

将修改后的非阻塞属性放入文件描述符中

fcntl(fd,F_SETFL,statu);

删除非阻塞模型

statu &= ~O_NONBLOCK;

（3）信号异步模型

使用信号去驱动函数

将文件IO（网络套接字）与IO信号绑定在一起

设置文件属性—>在文件操作过程中修改文件属性

#include <unistd.h>

#include <fcntl.h>
int fcntl(int fildes, int cmd, …);
fildes —->被操作的文件描述符
cmd —->操作命令
F_GETOWN ：获取当前进程所在的进程组
F_SETOWN ：设置当前进程所在的进程组
… —->具体的命令方式，根据cmd的参数而改变
getpid() :获取自己的进程号

设置IO信号响应—>只要接收到SIGIO，就会执行func函数

signal(SIGIO,func);

设置当前进程的进程组为自己—>SIGIO会影响到当前进程组的所有进程文件SIGIO
设置进程组为自己，保证当前进程组只有自己一个人

fcntl(sockfd,F_SETOWN,getpid());

设置文件IO为信号异步驱动模式—>设置sockfd产生变化，就相当于接收到了一个SIGIO的信号
注意的是，不会变成其他信号

fcntl(sockfd,F_SETFL,O_ASYNC);

（4）select模型

将文件描述符统一放在一个文件集合中，并设置超时时间。
如果超时时间未到达，那么等待并阻塞
如果超时时间到达，直接结束函数，继续运行
如果接到数据(可读，可写，可执行，出错)，不管超时时间是否到达，都立即往下运行

int select(int nfds, fd_set *restrict readfds,
fd_set *restrict writefds, fd_set *restrict errorfds,
struct timeval *restrict timeout);
nfds —->文件描述符的个数，从0开始计数
所以需要赋值为当前检测的文件描述符最大的数值+1
readfds —->读操作文件集合
writefds —->写操作文件集合
errorfds —->出错文件集合
timeout —->超时时间

void FD_CLR(int fd, fd_set *fdset);//从文件集合中删除某一个文件描述符
int FD_ISSET(int fd, fd_set *fdset);//判断某一个文件描述符是否还在文件集合中
void FD_SET(int fd, fd_set *fdset);//向文件集合中添加某一个文件描述符
void FD_ZERO(fd_set *fdset);//清空文件集合
fd —–>被操作的文件描述符
fdset —–>被操作的文件集合

超时时间结构体

struct timeval {
long tv_sec; 单位：秒
long tv_usec; 单位：微秒
};

系统编程-通信方式

2019-09-16

线程

进程间通信需要系统支持，如果一个进程内部就可以同时进行多个动作，
把同时进行多个动作的行为称为线程
线程—>资源调度的最小单位

如果虚拟空间有足够的开销，可以同时启用多个线程，那么就使用线程
如果不足，那么就使用进程。

创建线程

#include <pthread.h>
int pthread_create(pthread_t thread, const pthread_attr_t *attr,
void *(start_routine) (void ), void *arg);
pthread_t *thread —>线程号
const pthread_attr_t *attr —>线程属性：默认设置为NULL
void *(start_routine) (void *) —>线程需要执行的代码所在的函数
void *arg —>线程函数的参数

特别注意：线程库不是标准C函数库，而且也不是ubuntu的默认系统函数库
所以在使用线程时，需要链接线程库文件，加 -lpthread

gcc pid.c -o pid -lpthread

等待线程结束

#include <pthread.h>
int pthread_join(pthread_t thread, void *retval);
pthread_t thread —>需要等待的线程号
void *retval —>线程遗言，从pthread_exit()函数中获取
不需要遗言，设置为NULL

结束线程

#include <pthread.h>
void pthread_exit(void *retval);
retval —>线程的遗言，也就是给等待该线程结束的另一个
线程发送数据
不需要遗言，设置为NULL

笔试题：
main函数结束掉，线程一定结束掉，对吗？ 对，return或exit()会直接结束整个进程
主线程结束掉，线程一定结束掉，对吗？ 不对，主线程结束时使用pthread_exit()
子线程照旧进行
进程结束掉，线程一定结束掉，对吗？ 对

线程属性及传参

线程属性初始化

#include <pthread.h>

//摧毁线程属性
int pthread_attr_destroy(pthread_attr_t *attr);
pthread_attr_t *attr —->线程属性的变量

设置线程属性是否分离

#include <pthread.h>
int pthread_attr_setdetachstate(pthread_attr_t *attr, int detachstate);
pthread_attr_t *attr —->线程属性的变量
int detachstate —->PTHREAD_CREATE_DETACHED:分离属性
PTHREAD_CREATE_JOINABLE:非分离属性

pthread_attr_setXXXX —>设置线程属性
pthread_attr_getXXXX —>获取线程属性

线程传参

int num=12;
pthread_create(&tid,&attr,func,(void*)&num);

如果要传2个或2个以上的参数，必须使用结构体或数组或链表头节点传参

举例：
使用线程实现两个进程间的任意时刻的通信(消息队列)

取消线程

#include <pthread.h>
int pthread_cancel(pthread_t thread);
pthread_t thread —–>被取消的线程号

线程的同步与互斥

线程互斥锁：

在多个线程中使用互斥锁，同一时间只有一个线程

能够得到锁，其他线程会阻塞在加锁的位置，直到线程
解锁，其他线程中有一个可以开始运行

线程锁的man手册需要额外下载：
sudo apt-get install manpages manpages-dev manpages-posix-dev

#include <pthread.h>

创建并初始化互斥锁

int pthread_mutex_init(pthread_mutex_t *restrict mutex,

pthread_mutex_t *restrict mutex —->互斥锁的变量名
const pthread_mutexattr_t *restrict attr —->互斥锁属性，默认为NULL

销毁互斥锁

int pthread_mutex_destroy(pthread_mutex_t *mutex);
pthread_mutex_t *mutex —->互斥锁的变量名

#include <pthread.h>

加互斥锁—阻塞线程运行

int pthread_mutex_lock(pthread_mutex_t *mutex);

加互斥锁—检查互斥锁状态，返回0表示已经加锁，不会发生阻塞

int pthread_mutex_trylock(pthread_mutex_t *mutex);

解除互斥锁

int pthread_mutex_unlock(pthread_mutex_t *mutex);
pthread_mutex_t *mutex —->锁的名称

自旋锁：和互斥锁相同，但不会让线程进入睡眠状态，而是while(1)死循环不断调用CPU，

死锁

如果对加锁的程序进行了取消？
如果一个线程在加锁后的运行中，被取消线程，那么这个锁会一直
处于加锁的状态，影响后续程序的运行——>线程死锁

在任意一个加锁的线程中，都需要做死锁处理

#include <pthread.h>
//注册一个函数，当线程被取消的时候，执行该函数
void pthread_cleanup_push(void (routine)(void *),void *arg);
void (routine)(void *) —>被注册的函数
void *arg —>被注册的函数传入的参数

//写0，则表示删除注册的函数
void pthread_cleanup_pop(int execute);
int execute —>写0，则表示删除注册的函数

这两个函数一定是成对出现

当读锁有至少一个存在的时候，写锁是加不上去的
当写锁存在的时候，读锁和写锁都加不上去

posix匿名信号量

匿名信号量存在于栈里面

当信号量的值大于1的时候，可以进行P操作，也就是不会阻塞
当信号量的值等于0的时候，不可以进行P操作，也就是会阻塞

特别注意，当信号量的值<0时，P操作不可以进行，而且该线程/进程会被挂起到一个队列中
当进行多次V操作，使信号量大于0时，才会从第一个被等待的线程/进程开始运行

//创建匿名信号量并且赋初值

#include <semaphore.h>
int sem_init(sem_t *sem, int pshared, unsigned int value);
sem_t *sem —>匿名信号量变量名
int pshared —>写0表示该信号量是在线程间运行使用
unsigned int value —>该信号量的初值

信号量P操作—>申请资源，执行-1

#include <semaphore.h>
int sem_wait(sem_t *sem);
sem_t *sem —>信号量的指针

信号量V操作—>释放资源，执行+1

#include <semaphore.h>
int sem_post(sem_t *sem);
sem_t *sem —>信号量的指针

信号量运用在需要确定线程的运行先后顺序，并且交互使用的情况
互斥锁运用在线程资源不能同时使用的情况
读写锁运行在线程资源部分线程不能同时使用，部分线程需要同时使用的情况

条件变量

使用条件变量

ipc通信

ipcs -a 查看当前linux环境下所有的IPC通信情况

Shared Memory Segments：共享内存

共享内存

进程将自己内存的一部分与内核中的空间同步，读写操作都会被

同步到内核的共享内存空间，如果有另一个进程也同步了该空间，就
可以获取到相同的数据
共享内存中的数据与进程中的其他数据生存周期是一样的

创建键值

#include <sys/types.h>

#include <sys/ipc.h>
key_t ftok(const char *pathname, int proj_id);
const char *pathname —>共享内存所在的路径
int proj_id —>任意数值

要注意：两个使用相同共享内存的进程，参数必须是完全一样的
成功返回键值，失败返回-1

获取共享内存的ID号

#include <sys/shm.h>
int shmget(key_t key, size_t size, int shmflg);
key_t key —->键值
size_t size —->申请的共享内存大小，单位字节
int shmflg —->创建的模式
IPC_CREAT
成功返回共享内存ID号，失败返回-1

向内核申请共享内存，同时将内存首地址返回回来

#include <sys/types.h>

#include <sys/shm.h>

void shmat(int shmid, const void *shmaddr, int shmflg);
int shmid —->共享内存的ID号
shmaddr —->分配内存时的首地址
int shmflg —->共享内存的模式，一般写0表示可读可写
成功返回共享内存的首地址，失败返回(void) -1

释放申请的共享内存（只能释放当前进程的共享内存，内核中是释放不了的）
int shmdt(const void *shmaddr);

  • linux下只看共享内存的情况

    ipcs -m

  • linux下删除共享内存

    ipcrm -m +共享内存ID号

共享内存的接收与发送的内存是无关的，每一个进程都只对内核的共享内存负责

//对共享内存进行操作

#include <sys/ipc.h>

#include <sys/shm.h>

int shmid —->被操作的共享内存的ID号

IPC_RMID 删除共享内存
IPC_STAT 获取共享内存的信息
struct shmid_ds *buf—->执行其他动作需要传入的参数
删除动作时设置为NULL
获取动作时，存放数据的位置

信号的安全使用

1.如果父进程存在挂起的信号，则子进程不继承
2.如果父进程存在阻塞的信号，子进程会继承阻塞，将获取的信号挂起
3.信号是可以被嵌套的，但如果同时有多个信号被挂起，
可靠信号一定是响应优先级高的先响应
不可靠信号可能会被嵌套响应

不可靠信号最多挂起一个
可靠信号发送多少次，就挂起多少次

例子：假设信号挂起队列中存在10,12,35,36四个信号，按优先级
响应时36,35,12,10，当36响应的时候，10,12,35都还在队列中，
—>若此时发送信号10，由于不可靠信号最多挂起一个，所以36不会被嵌套
—>若此时发送信号35，由于可靠信号可以挂起多次，所以35运行多次

例子：假设信号挂起队列中存在10,35,36四个信号，按优先级
响应时36,35,10，当36响应的时候，10,35都还在队列中，
—>若此时发送信号12，信号12会嵌套在36响应过程中，但只能挂起一次

4.可以设置信号不被指定的信号嵌套，需要通过sigaction注册信号执行函数，
signal函数此时无效，同时要注意不能嵌套的信号不是被忽略，而是被阻塞，
当当前信号执行完毕时，仍然会再一次执行

5.信号是异步通信，信号响应的位置可能是程序的任意位置，那么如果
信号操作了全局变量，则很可能发生数值出错。
信号不可操作全局变量
库函数函数中也存在许多函数操作了全局变量，这些函数都是不安全函数
使用 man 7 signal可以查看安全函数

6.信号可以结束sleep和pause函数
信号可以结束进程的睡眠状态，所以也被称为软中断

信号挂起：
（1）进程在信号阻塞的情况下接到该信号
（2）进程在执行信号响应的情况下，再次接到该信号
信号阻塞：
进程接到信号不立即执行，要等待解除阻塞才执行
信号忽略：
进程屏蔽信号，不接受
信号响应：
进程接到信号后执行函数或缺省动作
信号嵌套：
进程在响应A信号的过程中，接收到B信号，会先执行B信号，再执行A信号
信号嵌套可以多次相互嵌套，但已响应的信号不能被再次嵌套，会被挂起
信号缺省动作：
执行linux系统指定的信号默认动作

目录操作

1.目录的打开 opendir()
 NAME   打开一个目录
   opendir, fdopendir - open a directory

SYNOPSIS
   #include <sys/types.h>
   #include <dirent.h>
   函数原型：
   DIR *opendir(const char *name);
   函数参数：
       const char *name  目录的完整地址
    返回值：
       成功: 返回DIR类型的指针指向你打开的目录
       失败：NULL
2.目录的读取 readdir
NAME 读取一个目录
   readdir, readdir_r - read a directory

SYNOPSIS
   #include <dirent.h>
   函数原型：
   struct dirent *readdir(DIR *dirp);
   函数参数：
         DIR *dirp 给我要读取的目录指针
   函数返回值：
       成功: 返回struct dirent *类型的指针指向你读的目录
       失败：NULL   
    struct dirent {
        ino_t          d_ino;       /* 节点编号 */
        off_t          d_off;       /* 偏移到下个节点 */
        unsigned short d_reclen;    /* 目录的长度 */
        unsigned char  d_type;      /* 文件的类型*/
        char           d_name[256]; /* 文件的名字 */
       };//存放的是目录中每一个文件的相关信息
    文件类型判断
   DT_BLK      This is a block device.      块设备文件

   DT_CHR      This is a character device.  字符设备文件

   DT_DIR      This is a directory.          目录文件

   DT_FIFO     This is a named pipe (FIFO).  管道文件

   DT_LNK      This is a symbolic link.      链接文件

   DT_REG      This is a regular file.       普通文件

   DT_SOCK     This is a UNIX domain socket. 套接字文件

   DT_UNKNOWN  The file type is unknown.     其他类型
3.关闭目录 closedir()
  NAME  关闭一个目录
   closedir - close a directory

 SYNOPSIS 
   #include <sys/types.h>
   #include <dirent.h>
   函数原型：
   int closedir(DIR *dirp);
   函数参数：
            DIR *dirp 要关闭的目录指针

4.创建目录 mkdir
  NAME  创建目录
   mkdir - make a directory

  SYNOPSIS
   #include <sys/stat.h>

   int mkdir(const char *path, mode_t mode);
   函数参数：
          const char *path   你要新建目录的位置
          mode_t mode        新建目录的权限
    返回值：
         成功：返回  0 
         失败：返回 -1

标准IO

2019-09-10
区别

系统IO: 常量操作
1. 直接调用系统函数接口
2. 不带IO缓冲区

标准IO:     指针操作
  1.  调用的标准库里面的函数
  2.  携带IO缓冲区
标准IO —>f开头的一系列函数
fopen()  fread() fwrite() fclose() fstat()
fseek()  ftell() ferror() feof() rewind()    
fprintf() fscanf() sprintf() sscanf()
fputs()   fgets()  fputc()  fgetc()
puts()  gets() getchar() putchar()

注意点：
系统IO,打开文件会得到文件描述符，文件操作围绕文件描述符

标准IO,得到文件指针，文件指针指向文件结构体，
       其中有个成员_fileno,这个就是文件描述符
